# TableContainer

import TableContainer from './Table.state';

## Basic
<TableContainer />

```jsx
  <TableContainer
    tableType='user'
    environment="client"
    // Data files
    tableData={tableData[currentPage]}

    onSearch={this.onClickPageChange}
    onRowClick={(id, rowValues) => console.log('id', id, 'rowValues', rowValues)}

    // pass from redux store
    page_count={Object.keys(tableData).length}
    total_count={12}

    // Top Bar (Optional Props)
    leftComponent={
      <DownloadButton />
    }
  />
```

## Props

| Property       | Type               | Required | Description |
| :-------       | :---               | :------- | :---------- |
| tableData      | `object`           | true     | Main table data, key should be the id of the item, entries should match the template types |
| tableType      | `string`           | true     | Defines the table columns and filters |
| environment    | `string`           | false    | defaults to 'admin', affects columns and filters |
| onSearch       | `function`         | true     | (pageNumber, filterValues, queryObject) the searching function |
| page_count     | `integer/string`   | true     | total number of pages |
| total_count    | `integer/string`   | true     | total number of object types |
| className      | `string`           | false    | className of input |
| unselectable   | `boolean`          | false    | determines if the table can be selectable |
| filterOptions  | `boolean`          | false    | provides option arrays for certain dropdown filters. See Readme for implemtation. |
| leftComponent  | `component`        | false    | component renders to the left of the search bar
| rightComponent | `component`        | false    | component renders to the right of the search bar
| customTemplate | `array of objects` | false    | custom template, follow the pattern defined on the repo
